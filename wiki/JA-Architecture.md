# コンパイラ構成

FAMI-C の処理パイプラインは次の通りです。

```text
C source -> lexer/parser -> AST -> 6502 assembly -> PRG ROM -> iNES ROM
```

## 字句解析と構文解析

コメントを取り除き、単純な `#define` を展開したあと、C ソースをトークン化します。再帰下降パーサーが宣言、文、式を AST に変換します。

## コード生成

生成されるコードは 8 ビットの 6502 ABI に合わせています。

- RAM 変数は `$0200` 以降に配置します。
- `$0100-$01FF` の 6502 スタックページを避けます。
- 関数引数は静的スロットで渡します。
- PPU とコントローラー操作はランタイム関数に分離しています。

## アセンブラと ROM

内蔵アセンブラがラベル、命令、`.byte`, `.word`, `.org` を解決し、最後に iNES ヘッダー付きの ROM を生成します。
