NES / Famicom 6502 Toolchain

FAMI-C

NES 및 패미컴용 6502에서 실행되는 프로그램을 C로 작성하고, 자체 컴파일러, 어셈블러, 런타임, iNES 패키저로 ROM까지 빌드합니다.

Quick Start Architecture 
python .\famic.py build .\examples\tetris.c `
  -o .\build\tetris.nes `
  --asm .\build\tetris.asm

Output: mapper 0 iNES ROM for Mesen, FCEUX, Nestopia, or hardware-compatible workflows.

Quick Start

Requirements

  • Python 3.10 or newer
  • Windows PowerShell, macOS shell, or Linux shell
  • NES emulator such as Mesen 2.2.1

Build Tetris

python .\famic.py build .\examples\tetris.c -o .\build\tetris.nes --asm .\build\tetris.asm

Windows users can also run:

.\build.ps1

Run Tests

python -m unittest discover -s tests

The tests verify compilation, PRG size, iNES header fields, reset vectors, and RAM placement.

Compiler Architecture

C sourceLexer and parser
AST8-bit NES-oriented semantic model
6502 assemblyStatic locals, native runtime calls
PRG ROMBuilt-in 6502 assembler
iNES ROMNROM-128 mapper 0 package

FAMI-C is intentionally self-contained. The compiler emits assembly, the assembler resolves labels and vectors, and the packager writes a 16 KB PRG + 8 KB CHR iNES image.

Supported C Model

Supported

  • char, unsigned char, int, unsigned int, void
  • 8-bit arithmetic, comparison, logical operators, arrays, and function calls
  • if, else, while, for, break, continue, return
  • Global RAM variables and const unsigned char ROM tables

NES Runtime API

extern void wait_vblank(void);
extern void ppu_put(unsigned char x, unsigned char y, unsigned char tile);
extern unsigned char read_pad(void);
extern unsigned char rand8(void);

Current Limits

  • Arithmetic ABI is 8-bit.
  • Locals are statically allocated; recursion is not supported.
  • Pointers, structs, casts, varargs, and libc are not implemented.
  • Generated ROMs target mapper 0 / NROM-128.

Example: Tetris

The example in examples/tetris.c demonstrates static arrays, ROM lookup tables, controller polling, collision checks, line clearing, and vblank-safe PPU updates.

Controls

  • Left / Right: move piece
  • Down: soft drop
  • A: rotate

Verified With

  • Mesen 2.2.1 on Windows
  • Automated Python unit tests
  • iNES header and vector checks

한국어 문서

FAMI-C 사용 방법

FAMI-C는 NES/패미컴 프로그램을 C로 작성하기 위한 작은 툴체인입니다. 목적은 최신 데스크톱 C 전체를 6502에 억지로 옮기는 것이 아니라, 실제 NES 개발에 필요한 정적 메모리, 8비트 연산, PPU/패드 런타임을 단순하게 제공하는 것입니다.

  1. 소스 파일을 examples 또는 별도 폴더에 작성합니다.
  2. extern 런타임 함수를 선언하고, main(void)을 구현합니다.
  3. python .\famic.py build source.c -o build/game.nes --asm build/game.asm으로 빌드합니다.
  4. Mesen 같은 에뮬레이터에서 .nes 파일을 엽니다.

日本語ドキュメント

FAMI-C の使い方

FAMI-C は NES / ファミコン向けの小さな C ツールチェーンです。 6502 に合わせた 8 ビット ABI、静的メモリ、PPU 更新、コントローラー入力を シンプルな C サブセットから扱えるようにします。

  1. C ソースに main(void) を実装します。
  2. 必要なランタイム関数を extern 宣言します。
  3. python .\famic.py build source.c -o build/game.nes --asm build/game.asm を実行します。
  4. Mesen などのエミュレーターで生成された ROM を開きます。

English Documentation

How To Use FAMI-C

FAMI-C is a compact C toolchain for NES / Famicom programs. It favors a practical 8-bit programming model over full hosted ISO C, making it suitable for small mapper-0 games and demos.

  1. Write a C source file with main(void).
  2. Declare the native runtime functions your program uses.
  3. Run python .\famic.py build source.c -o build/game.nes --asm build/game.asm.
  4. Open the generated ROM in Mesen or another NES emulator.
