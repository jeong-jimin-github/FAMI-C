# 컴파일러 아키텍처

FAMI-C의 빌드 흐름은 다음과 같습니다.

```text
C source -> lexer/parser -> AST -> 6502 assembly -> PRG ROM -> iNES ROM
```

## 1. 전처리와 토큰화

`famic.py`는 C 소스에서 주석을 제거하고, 단순한 객체형 `#define`을 확장합니다. 이후 키워드, 식별자, 숫자, 연산자를 토큰으로 만듭니다.

## 2. 파서

재귀 하강 파서가 전역 선언, 함수 선언, 문장, 식을 AST로 변환합니다. 연산자 우선순위는 별도 테이블로 관리합니다.

## 3. 코드 생성

코드 생성기는 8비트 누산기 중심의 6502 코드를 만듭니다.

- 전역 변수와 로컬 변수는 NES 내부 RAM `$0200` 이후에 배치합니다.
- `$0100-$01FF` 스택 페이지는 침범하지 않습니다.
- 함수 인자는 정적 슬롯으로 전달합니다.
- PPU와 컨트롤러는 네이티브 런타임 함수를 호출합니다.

## 4. 어셈블러

내장 어셈블러가 라벨, `.byte`, `.word`, `.org`, 상대 분기, 절대 주소를 처리합니다.

## 5. ROM 패키징

최종 출력은 iNES 헤더, 16 KB PRG ROM, 8 KB CHR ROM으로 구성됩니다.
